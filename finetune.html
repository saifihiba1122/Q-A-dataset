The technique

QLoRA: a 7B model on a 16 GB GPU.

Full fine-tuning of a 7B model needs multiple data-centre GPUs. Two ideas collapse that to a single free card.

Quantization

4-bit frozen base

The model's billions of weights are compressed from 16-bit to 4-bit and frozen. That alone cuts memory roughly 4×, which is what lets a 7B model load inside a free Colab T4.

LoRA

Train tiny adapters

Instead of updating the frozen weights, small low-rank adapter matrices are trained alongside them. Only 40.4M of 7.66B parameters (0.53%) ever change — the adapter is a few megabytes, not gigabytes.

SFT

Supervised fine-tuning

The model learns from (question → ideal answer) pairs, formatted with Qwen's chat template so it answers in the right structure. Pure imitation of the curated dataset.

Efficiency

Gradient tricks

Gradient accumulation simulates a larger batch (2 × 4 = 8) within tight memory, and gradient checkpointing trades a little compute for a lot of saved VRAM. Together they keep the whole run on one card.

What the curve shows

Loss fell, then settled.

Training loss dropped sharply in the first ~10 steps as the adapter learned the dataset's format, then settled into the 0.7–1.0 band — a healthy curve for a small, focused dataset over two epochs.

Read it right

Loss ≠ accuracy

A falling loss means the model is fitting the data — it is not a percentage score. Real quality is judged separately by prompting the model and, later, structured evaluation.

Honest scope

225 examples

This was a deliberately small dataset to prove the pipeline end-to-end. The biggest future gain comes from more curated data, not from tweaking hyperparameters.
